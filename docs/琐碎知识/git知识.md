## 以前已经学过一些基础的了 这里是放实际遇到的问题
1. 推送到远程： 
    ~~~text
   先创建远程仓库 复制ssh  
   然后 推送的时候定义远程仓库
    ~~~
2. 定义远程仓库时 的名称origin
    ~~~text
   因为可以本地 1 对 多个远程 所以得给他们起名字以区分
   默认origin
   ~~~
3. 不要远程仓库了：
    ~~~text
   先控制台 取消关联
   git remote remove origin
   
   然后删除github 库就行
    ~~~
4. 本地修正提交 再推；看来确实是不能实现 不增加提交记录了
    ~~~text
   正常的思维：就是推到远程仓库的流程
   0. 本地完成修改
   1. 拉取远程仓库的最新
   2. 看有没有与本地的冲突；没有就推；有就得合并 取舍（或者变基）
   （所谓冲突 应该是一个具体文件 内容不一样； 如果是多一个新文件 应该不是冲突）
   ~~~
   
5.   arg
6.  erg
